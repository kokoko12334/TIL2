

sync async

sync는 동기 => 프로그램의 실행이 멈추거나 하는 경우가 있음=> 다른 작업을 수행가능하지 않음 => "실행 흐름"이 끊김 프로그래밍 방식

async 는 비동기 => 프로그램의 흐름이 끊기지 않고 다른 작업 수행가능 => "실행 흐름"이 끊기지 않음 프로그래밍 방식



blocking => 입출력 처리 장치에서 처리 결과를 기다릴때까지 대기
non-blocking => 입출력 처리장치에서 처리 결과를 기다리지 않고 다른 작업이 있으면 수행=> 이후에 I/O작업이 완료되면 callback 함수나 polling으로
처리 받은 결과값을 가지고 후속작업을 진행



blocking io라고 해도 이 blocking io가 별도의 다른 채널, 쓰레드에서 작업을 수행(병렬)을 해서 전체 실행 흐름에 영향을 안준다면(멈추지 않음)
